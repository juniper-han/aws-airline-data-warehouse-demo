# 整体架构设计
## 项目概述
本项目基于 AWS 云服务 + Apache Kafka 搭建流批一体数据湖仓解决方案，覆盖实时消息队列、数据湖存储、ETL 集成、数仓建模、任务编排、自动化运维、监控告警全链路，为云原生大数据解决方案提供标准化落地实践。

## 技术组件清单
### 实时链路
- AWS MSK（托管 Kafka）：实时事件采集、消息缓冲、业务解耦
- Lambda：轻量数据消费、事件触发、自动化接口

### 数据湖层
- S3：离线&实时增量数据统一存储
- Lake Formation：数据湖权限管控、元数据管理、访问隔离

### 数据集成层
- AWS Glue：全托管 ETL，完成数据清洗、转换、批量调度

### 数仓存储层
- Redshift：云端数仓分层建模、多维分析
- DynamoDB：高并发维度表、配置数据、字典存储

### 调度与运维
- Step Functions：多组件工作流编排
- CloudWatch：全链路监控、日志采集、延迟观测
- SNS：异常告警、任务失败通知

## 核心数据流
1. 实时业务事件 → MSK Kafka 消息队列生产入队
2. 消费端将流式数据落地至 S3 数据湖
3. Lake Formation 统一数据权限与元数据治理
4. Glue 完成数据清洗、标准化、脏数据过滤
5. 加工后数据分层写入 Redshift 数仓
6. 关联 DynamoDB 维度表补全业务字段
7. Step Functions 串联全流程自动化调度
8. CloudWatch 监控链路健康，异常通过 SNS 推送告警

## 架构优势
1. 流批统一：实时 Kafka 流数据与离线批量数据统一入湖入仓
2. 云原生托管化：降低运维成本，轻量化部署
3. 模块化解耦：各组件职责清晰，独立扩展
4. 可落地：基于 AWS 免费套餐搭建，支持本地验证与企业迁移
