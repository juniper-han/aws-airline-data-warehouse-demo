# 数据建模设计文档
## 设计原则
1. 分层解耦：采用 ODS / DWD / DWS 标准三层数仓架构
2. 维度建模：基于星型模型设计，适配多维分析场景
3. 流批统一：实时与离线链路指标口径、维度逻辑完全一致
4. 云适配设计：兼顾查询性能与存储成本，适配云数仓特性

## 数仓分层说明
### ODS 原始层
存储原始日志、Kafka 落地原始数据，保留全量字段，用于数据溯源与重跑。
### DWD 明细层
完成数据清洗、格式统一、空值过滤，保留业务明细，关联基础维度信息。
### DWS 汇总层
基于多维度聚合计算周期指标，减少重复计算，支撑快速业务分析。

## 数据表设计
### 事件事实表
存储用户行为、业务事件明细，数据来源为 Kafka 实时流与离线批量数据。
### 维度表
包含用户、地区、设备等字典数据，采用 DynamoDB 高速查询 + Redshift 离线分析双存储方案。

## 数据质量规范
1. 主键唯一约束，杜绝数据重复
2. 时间字段统一标准化，保证流批时间对齐
3. 核心字段非空校验，统一隔离脏数据
4. 配置数据生命周期策略，实现冷热分层存储
